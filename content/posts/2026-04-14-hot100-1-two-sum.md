## 核心思路

用哈希表一边遍历数组，一边记录已出现的数字和下标。

遍历到 `nums[i]` 时，查哈希表中是否存在 `target - nums[i]`：

- 存在：找到答案，返回两个下标
- 不存在：把当前数字和下标存入哈希表

## 代码

```cpp
class Solution {
    unordered_map<int, int> ans;
public:
    vector<int> twoSum(vector<int>& nums, int target) {
        for (int i = 0; i < nums.size(); i++) {
            auto it = ans.find(target - nums[i]);
            if (it != ans.end()) {
                return {it->second, i};
            }
            ans[nums[i]] = i;
        }
        return {};
    }
};
```

## 为什么先查再存

题目不能使用同一个元素两次。如果先存再查，当前元素可能和自己配对。

## 复杂度

- 时间：O(n)
- 空间：O(n)

## 易错点

1. 哈希表存的是「数字 -> 下标」，不是「下标 -> 数字」
2. 必须先查补数，再存当前数字
3. 题目保证有解，找到后直接返回
