题目代码：

```cpp
class Solution {
    unordered_map<int, int> ans;
public:
    vector<int> twoSum(vector<int>& nums, int target) {
        for (int i = 0; i < nums.size(); i++) {
            auto it = ans.find(target - nums[i]);
            if (it != ans.end()) {
                return {it->second, i};
            }
            ans[nums[i]] = i;
        }
        return {};
    }
};
```

## 核心思路

用哈希表一边遍历数组，一边记录已经出现过的数字和它的下标。

遍历到 `nums[i]` 时，先去哈希表里查 `target - nums[i]` 是否存在：

- 如果存在，说明找到了两个和为 `target` 的数，直接返回答案。
- 如果不存在，就把当前数字和下标存进哈希表，供后面的元素使用。

## 为什么这样做

这题本质上是在问：

对于当前数字 `nums[i]`，前面是否已经出现过一个数，刚好等于 `target - nums[i]`。

如果每次都在线性查找前面的数，时间复杂度会是 `O(n^2)`。

哈希表能把“某个数是否存在”的查找优化到平均 `O(1)`，所以整体只需要遍历一次数组。

## 解题步骤

1. 定义哈希表，键是数字，值是这个数字对应的下标。
2. 从左到右遍历数组。
3. 对于每个 `nums[i]`，先查 `target - nums[i]` 是否已经在哈希表中。
4. 如果在，直接返回这两个数的下标。
5. 如果不在，再把 `nums[i]` 和 `i` 存进去。

## 为什么一定要先查再存

因为题目不能使用同一个元素两次。

如果先存再查，当前元素可能会和自己配对，产生错误答案。

所以必须先检查前面有没有补数，再存当前值。

## 复杂度

- 时间复杂度：`O(n)`
- 空间复杂度：`O(n)`

## 容易错的点

1. 哈希表里存的是“数字 -> 下标”，不是“下标 -> 数字”。
2. 一定是先查补数，再存当前数字。
3. 题目一般保证有解，所以找到后可以直接返回。

## 一句话记忆

看到“在数组里找两个数凑成目标值”，优先想到哈希表配对。
